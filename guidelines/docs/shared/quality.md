# Code Quality

- All new code must have associated **unit tests**.
- Code must pass the project's **linter** with zero violations.
- The project must **build successfully** after changes.
- Test coverage threshold is **80%** for new code.
- Follow existing patterns and conventions in the codebase — consistency over personal preference.
- Component organization: services in `src/app/shared/services/`, components in `src/app/features/`, models in `src/app/models/`.
